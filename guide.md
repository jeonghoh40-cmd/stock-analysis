# AI 주식 스크리닝 v4 — 운영 가이드

## 1. 실행 방법

```bash
# Claude에게 요청
"전체 분석 실행해줘"

# 직접 실행 (배치)
run_all_v4.bat

# 직접 실행 (Python)
python -c "import stock_advisor_v4; stock_advisor_v4.main()"

# 자동 스케줄 (Task Scheduler 등록)
run_daily.bat  →  logs/daily_YYYYMMDD.log 저장
```

## 2. 실행 순서 (고정)

```
STEP 0  캐시 정리
        - Claude 캐시 7일 이상 → 자동 삭제
        - 스크리닝 캐시 3일 이상 → 자동 삭제

STEP 1  기술적 스크리닝  [① 전 종목]
        - 대상: KOSPI ~222개 / KOSDAQ ~168개 / US 83개 (총 ~473개)
        - 지표: RSI · MACD · Stochastic · ADX · OBV · ATR · 52주위치
        - 당일 캐시 있으면 재사용 (yfinance 재수집 생략)

STEP 2  펀더멘털 보강  [② 상위 후보]
        - 매수 후보 상위 10개 대상 (시장별)
        - P/E · P/B · ROE · 펀더멘털 점수 반영

STEP 3  보조 데이터 수집  [③ 거시·심리·섹터·뉴스] — 병렬 수집
        - 거시지표: USD/KRW · WTI · VIX · US10Y · KOSPI · 나스닥 등 11개
        - Fear & Greed Index (CNN)
        - 섹터 ETF 흐름 15개
        - 해외 주요 지수 스냅샷
        - 최신 뉴스 10건

STEP 4  외부 변수 + 투자자 포트폴리오 + 테마
        - 외부 변수 로드 (external_events.json)
        - 유명 투자자 포트폴리오 수집 → 매칭 종목 +5 가산 → 풀 재정렬
          (Pelosi, ARK, 박세익, 존리, 이채원, 김민국, 강방천)
        - 국내 테마별 상위 종목 추출 (반도체·2차전지·바이오 등)

STEP 5  Claude 심층 분석  [④]
        - 입력: 확대 후보 (KOSPI 10 / KOSDAQ 6 / US 10)
                + 거시·뉴스·섹터·외부변수·투자자포트폴리오·테마
        - 출력: JSON(top_buy/top_sell 순위) + 고정 포맷 분석 리포트
        - JSON이 최종 선별 순서를 결정 (Claude 추천 우선)
        - 당일 캐시 있으면 재사용

STEP 6  최종 선별 (Claude JSON 순위 우선)
        - 매수 TOP: KOSPI 5개 / KOSDAQ 3개 / US 5개
        - 매도 TOP: KOSPI 3개 / KOSDAQ 2개 / US 3개
        - Claude top_buy/top_sell 순서 우선, 미추천 슬롯은 점수 순 채움

STEP 7  리포트 저장  [⑤]
        - report_v4.txt  (plain text 이메일 본문)
        - report_v4.html (HTML 보관용)

STEP 8  성능 추적  [⑤]
        - stock_performance.db 에 추천 종목 저장
        - 30일 성과 요약 → performance_report.txt

STEP 9  알림 발송  [⑦]
        - 이메일: geunho@stic.co.kr  (plain text)
        - 텔레그램: 설정 시 발송 (현재 미설정)
```

## 5. 주요 설정값 (stock_advisor_v4.py)

| 항목 | 값 | 위치 |
|------|-----|------|
| 매수 추천 수 | KOSPI 5 / KOSDAQ 3 / US 5 | `RECOMMEND_COUNT` |
| 매도 추천 수 | KOSPI 3 / KOSDAQ 2 / US 3 | `SELL_COUNT` |
| Claude 모델 | claude-sonnet-4-6 | `ask_claude_v4()` |
| Claude max_tokens | 5,000 | `ask_claude_v4()` |
| 프롬프트 상한 | 28,000자 (≈8,000 토큰) | `_MAX_PROMPT_CHARS` |
| 스크리닝 캐시 유효 | 당일 (3일 초과 자동 삭제) | `_clean_old_screening_caches` |
| Claude 캐시 유효 | 7일 초과 자동 삭제 (MD5 파일만 대상) | `clear_old_cache(7)` |
| 이메일 형식 | plain text | `send_email()` |

## 6. 외부 변수 관리 (external_events.json)

지정학·정책 이벤트를 수동으로 추가해 Claude 분석에 반영합니다.

```bash
# 추가
python external_events.py add

# 현재 목록 확인
python external_events.py list

# 삭제
python external_events.py remove <id>
```

형식:
```json
[
  {
    "category": "지정학",
    "title": "이란-미국 전쟁 — 이란 자멸 시나리오",
    "impact": "매수",
    "added": "2026-03-16"
  }
]
```
