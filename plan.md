### ETF 전용 필터 엔진 (`etf_filter.py`)

> 개별 종목 스코어링과 **완전히 분리된 별도 엔진**으로 ETF만 처리한다.
> ETF에는 개별 종목 기준(P/E, EPS 등 펀더멘털)을 적용하지 않는다.

```
개별 종목 엔진 (stock_advisor_v4.py)
  └─ 기술적(40) + 펀더멘털(25) + 수급(20) + 매크로(15)

ETF 전용 엔진 (etf_filter.py)          ← 별도 모듈
  └─ 유동성(30) + 기술적추세(30) + 테마강도(25) + 괴리율(15)
       ↓
  두 엔진 결과를 합산 후 TOP 10 선별
  (개별 종목 후보 + ETF 후보 통합 랭킹)
```

| ETF 스코어 항목 | 배점 | 세부 기준 |
|---------------|------|----------|
| 유동성 | 30점 | 거래대금·AUM·NAV 괴리율 |
| 기술적 추세 | 30점 | RSI·MACD·이동평균 (개별 종목과 동일 로직) |
| 테마 강도 | 25점 | 섹터 ETF 자금 흐름 (`etf_analysis.py`) |
| 괴리율·추적오차 | 15점 | NAV 대비 할인 시 가산점, 프리미엄 시 감점 |

| 목표가 도달 | ATR 기반 동적 목표가 (진입가 + ATR × 2.5) | Claude AI 생성 |

### 포지션 관리

| 구분 | 기준 |
|------|------|
| 단일 종목 최대 비중 | 계좌의 5% 이하 (HARD-GATE) |

### 리스크 계층 구조

## 📊 Phase 2.6 — 백테스팅 엔진 (Backtesting)

### 구현 목표

### 사용할 라이브러리

| 라이브러리 | 용도 | 설치 |
|-----------|------|------|

### 백테스트 데이터 원칙

#### Point-in-Time (PIT) 데이터

> 백테스트 시점에 **실제로 알 수 있었던 데이터만** 사용한다. 미래 데이터 참조(Look-ahead Bias)를 완전히 차단한다.

| 데이터 유형 | PIT 처리 방법 |
|------------|-------------|
| 재무 데이터 (P/E, EPS 등) | 공시 발표일 기준으로 적용 (분기보고서 제출일 이후) |
| 유니버스 구성 | 해당 시점의 지수 편입 종목 기준 (현재 편입 종목 기준 X) |
| 기술적 지표 | 해당 일자까지의 과거 데이터만 사용 |
| ETF NAV·괴리율 | 해당 일자 장 마감 기준 |

#### Delisted(상장폐지) 종목 포함

> 생존편향(Survivorship Bias)을 제거하기 위해 테스트 기간 내 상장폐지된 종목을 반드시 포함한다.

| 처리 방법 | 상세 |
|----------|------|
| 상폐 종목 데이터 소스 | yfinance 과거 데이터 + 별도 상폐 종목 DB 구축 |
| 상폐 발생 시 처리 | 상폐일 종가를 청산가로 적용 (-100% 또는 실제 청산가) |
| 거래정지 종목 | 정지 기간 수익률 = 0으로 처리 |
| 관리종목 편입 시 | 편입 즉시 포지션 50% 강제 청산 가정 |

### 백테스트 산출 지표

| 지표 | 산출 방법 | 합격 기준 |
|------|----------|----------|
| 누적 수익률 | 추천 종목 동일 비중 매수 후 N일 보유 | 벤치마크(KOSPI/S&P500) 대비 양수 |
| 승률 | (수익 종목 수 / 전체 추천 수) × 100 | **≥ 55%** |
| MDD (최대 낙폭) | 고점 대비 최대 하락률 | **≤ 10%** |
| 샤프 지수 | (연간 수익률 − 무위험 수익률) / 연간 변동성 | ≥ 1.0 |
| 평균 보유 기간 수익률 | 진입~청산 평균 수익률 | ≥ 3% |

### 예상되는 사이드 이펙트

| 위험 | 영향도 | 대응책 |
|------|--------|--------|
| Look-ahead Bias — 미래 재무 데이터 참조 | 높음 | 공시일 기준 PIT 처리 강제, 단위 테스트로 검증 |
| 생존편향 — 현재 존재하는 종목만 테스트 | 높음 | 상폐 종목 DB 구축 및 포함 (필수) |
| 과적합 (Overfitting) | 높음 | Out-of-sample 구간 별도 검증 (훈련 70% / 검증 30%) |
| 수수료·슬리피지 미반영 | 중간 | 편도 0.015% 수수료 + 0.1% 슬리피지 가정 적용 |
| 상폐 종목 데이터 부재 | 중간 | yfinance 제공 범위 내 수집, 누락 시 명시적 경고 출력 |

### 체크리스트

- [ ] `backtest.py` 작성 (기간 입력 → 성과 지표 출력)
- [ ] Point-in-Time 처리: 재무 데이터 공시일 기준 적용
